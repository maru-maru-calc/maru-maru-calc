# まるまるでんたく Depth Path リデザイン案

## 方針

Duolingoから借りるのは、色やキャラクターではなく **学習が進んでいく気持ちよさ**。

- 縦に進むレッスンパス
- 次に押す場所がすぐわかる大きな現在地
- 完了、次、未開放の状態差
- ただし画面内の説明文は極力置かない

見た目の主役は、まるまるでんたく固有の **泡、まる、まとまり**。鳥、緑のブランドカラー、Duolingo固有のアイコン配置は使わない。
小さい子どもや他言語のユーザーでも触れるように、数字、記号、深度、状態差で意味を伝える。

## Launch / start screen

アプリ起動直後は、説明を詰め込みすぎず、計算フィールドの世界へ入る入口にする。

- 目的
  - `●● calc.` の名前を最初に強く見せる
  - 泡、まる、水面の世界観を一目で伝える
  - ゲーム画面と同じ操作フィールドを見せて、すぐ遊べそうな印象にする
- 表示
  - 上部にタイトル `●● calc.`
  - 下部に、四則演算タブと同じ白い浮きパネルのテイストで `8 +` を控えめに表示する
  - ゲーム画面と共通の演算タブ、待機泡、水面カーブを置く
  - 起動画面の演算タブは `+`、`−`、`×`、`÷` の記号だけにする
  - 起動画面上部の待機泡は、2の泡を1つだけ置く
  - フィールド内には、8を表す1のまるを8個置く
  - 1のまるは小さめにし、フィールド底のカーブに接するように並べる
  - 8個は一列に詰めすぎず、左3個、右5個のように自然なまとまりにする
  - 起動画面では、フィールド内のまるに数字を入れない
  - 補助コピーや操作説明は置かない
  - `やりなおす` のような文字ボタンは置かない
  - 画面遷移を促す導線として、式表示と同じ淡い白パネル調の文字なし角丸四角ボタンを置く
  - ページインジケータ風のドットは置かない
- レイアウト
  - 上部にタイトルを置き、ロゴや水面装飾と重ねない
  - 中央から下はゲーム画面と同じフィールド構成にする
  - 起動画面のフィールド底は画面下に寄せ、左右にはみ出す大きな円弧で緩やかに見せる
  - 背景の泡は少数にして、初回画面の静けさを保つ
  - 端末下部には十分な余白を残す

## Visual Concept

テーマ名: `Depth Path`

海の中を少しずつ深く潜りながら、まるを大きなまとまりへ育てていく。

- パス: 点線や道路ではなく、水中の深度軸
- ステージ: Launch/Gameの数字泡と同じ、白く透ける泡ノード
- 現在地: 輪郭を強めた大きな泡 + 中央の目標値
- 完了: 泡の右上に小さな星
- 未開放: くもった泡、または泡の中の小さな鍵
- 進行: WorldSelectからStageSelectへ、選択が進むほど深く潜る

## WorldSelect / operation routes

四則演算の入口は、ステージパスとは別の選択画面に分ける。画面内に日本語の説明名は置かず、`+`、`−`、`×`、`÷` の泡ノードで表現する。浅いところから深いところへ潜るほど、学習領域が進む。

- 目的
  - 子どもや他言語ユーザーが、文字を読まずに演算を選べる
  - 四則演算の選択を、リストではなく海の深度として見せる
  - WorldSelectで選んだ深度から、StageSelectへ自然につながる
- 表示
  - 上から順に `+`、`−`、`×`、`÷` の泡ノード
  - 選択中は大きいリングと強い水色
  - ノード本体はLaunch/Gameの泡と同じ透明感、光、内側の白い反射を使う
  - クリア済みは泡の色を変えず、右上の星で示す
  - 進捗は左の深度メーターで控えめに示す
  - 未開放はくもった泡
  - 画面内の説明文、島名、地名は置かない
  - ノード上に数量バッジは置かない。ゲーム画面の泡の数量表現と混同させない
- レイアウト
  - 海面から深海へ向かう縦の深度軸
  - ノード自体を入口にする
  - 下部の決定ボタンは置かない

## StageSelect / depth path

- WorldSelectで選んだ演算の中に入り、同じく深度を下げながらステージを進む
- 画面内の日本語ラベル、吹き出し、下部タブは置かない
- WorldSelectへ戻れるように、左上に淡い白パネル調の戻るボタンを置く
- ステージ番号は各泡ノードの左上に置き、テーマをまたいでも `#1`、`#2`、`#3` と通し番号にする
- ノード右下に数量バッジは置かない。ゲーム画面の泡の数量表現と混同させない
- ステージ構造は一直線ではなくツリーにする
- 例: `10 #1` をクリアすると、同じテーマの `10 #2` と、次テーマの `20 #3` の両方が開く
- `10` 系を全部終えないと `20` に進めない、という強い依存にはしない
- 中央
  - 海面から深海へ向かう淡い水色グラデーション
  - 大中小の装飾泡で奥行きを作る
  - 深度軸から左右に泡ノードが枝分かれする
  - 現在地は一番大きく、中央の数字で示す
- ノード状態
  - `active`: 透明な大泡 + 濃い水色リング + 中央に目標値
  - `done`: 通常の透明泡 + 右上の小さな星
  - `locked`: くもった白泡 + 小さい鍵
- 進捗
  - 左の深度メーターで控えめに示す
  - テキストの進捗表記は置かない

## Screen Order

- `Launch / start screen`
- `WorldSelect / operation routes`
- `StageSelect / depth path`
- `Game / current direction`

## Current implementation audit

現行の `src/app/index.tsx` と `src/components/MarumaruGame.tsx` は、次の状態になっている。

- 起動後はWorldSelectを経由せず、演算タブとステージ一覧が同じ画面にある
- ステージ泡は完了、次、待機の見た目を持つが、待機中も押してゲームを開始できる
- 進捗は画面内のReact stateに保存され、アプリ再起動後には残らない
- ゲームの戻る操作はステージ一覧へ戻る
- すべての泡を使い切り、値が目標と一致すると約650ms後にクリアになる
- クリア時は盤面上に結果パネルが重なり、再試行と次のステージを選べる
- 目標に一致せず、これ以上まとめられない場合は失敗パネルが重なり、再試行できる
- 次のステージは配列上の次要素で、ツリー分岐には対応していない

旧実装には永続化、最後に遊んだステージ、クリア済み、次のおすすめステージの概念がある。今後の実装では、この進捗管理をツリー構造へ合わせて再利用する。

## Interaction / transition patterns

### Normal route

1. Launchの右下ボタンを押すとWorldSelectへ進む
2. WorldSelectの開いている演算泡を押すと、その演算のStageSelectへ進む
3. StageSelectの開いているステージ泡を押すとGameへ進む
4. Game左上の戻るボタンは、直前のStageSelectへ戻る
5. StageSelect左上の戻るボタンは、直前のWorldSelectへ戻る

戻る操作は履歴を保ち、選択していた演算とスクロール深度を維持する。

### Locked node

- 鍵付き泡を押しても画面遷移しない
- 泡を短く左右に揺らす
- 解放条件になっている直前ノードと接続線を一度だけ淡く光らせる
- 日本語の説明ダイアログは出さない

### Game clear

- 盤面を消さず、その上に半透明の結果レイヤーを重ねる
- 中央に星と完成した式を表示する
- 主操作は次の問題へ進む再生記号
- 補助操作は再試行記号とStageSelectの枝記号
- クリア情報を保存してから遷移可能にする

### Branch unlock

- `#1` をクリアした場合、StageSelectへ戻ると `#1` の泡右上に星が付く
- 同時に開く `#2` と `#3` の接続線を明るくし、両方の泡を短く発光させる
- 自動で一方へ遷移せず、次に遊ぶ枝を選べる状態にする
- Gameの次ボタンを押した場合は、解放された候補のうち同じテーマ側を優先する

### Game retry

- 目標に一致しなかった場合も盤面上に結果レイヤーを重ねる
- 強い赤色や失敗を責める文章は使わず、`8 ≠ 10` のように現在値と目標値を示す
- 主操作は再試行
- 補助操作としてStageSelectへ戻れる

## Game / current direction

Game画面はLaunch画面と同じ構成をベースにする。違いは上部タイトル、戻るボタン、下部のやり直し導線だけにする。

- ヘッダー
  - 式表示や下部ボタンと同じ淡い白パネル調の戻るボタンを置く
  - Launchの `●● calc.` の位置を `●● = 10` に差し替える
- 演算タブ
  - Launchと同じ位置、同じ見た目
  - `+`、`−`、`×`、`÷` の記号だけにする
- 待機泡
  - Launchと同じく、2の泡を1つだけ置く
- フィールド
  - Launchと同じ水面カーブ、背景泡、8個の1まる配置を使う
- 下部
  - Launchと同じ `8 +` の式表示
  - 戻るボタンと混同しないように、右下にはリトライを示す回転矢印系の角丸四角ボタンを置く
  - 右下ボタンは式表示と同じ淡い白パネル調にする
  - ページインジケータ風のドットは置かない

Depth PathのテイストはWorldSelectとStageSelectに強く使う。LaunchとGameでは「まるを出す、落ちる、まとまる」が最優先。

### 実機スケールの目安

390px幅のスマートフォン画面では、相談用モックでも以下を上限にする。

| 要素 | 目安 |
| --- | ---: |
| ゲーム画面タイトル | 32〜34px |
| ステージ補助ラベル | 14〜15px |
| 演算タブ高さ | 52〜56px |
| 演算記号 | 24〜26px |
| 待機泡 | 52〜58px |
| 待機泡内の1まる | 17〜20px |
| 数量バッジ | 24〜28px |
| 下部メッセージ | 18〜20px |
| 下部Safe Area余白 | 20〜28px |
| フィールド開始位置 | 300〜320px |

ステージ選択の現在地ノードだけは、押す場所を明確にするため100px前後まで大きくしてよい。ゲーム画面では、待機泡と文字を大きくしすぎるとフィールドの主役感が落ちる。待機泡は10個前後並ぶ前提なので、見た目は小さめにする。
下部の式表示と `やりなおす` は画面端に接触させず、最低20px以上の余白を残す。

## Duolingo-like But Not Duolingo

使う:

- 進行パスのわかりやすさ
- 押すべき場所の強い視認性
- 完了/次/未開放の状態差
- 画面内の楽しさと達成感

使わない:

- 明るいグリーン基調
- 鳥のマスコット
- 盾、炎、宝箱などの固有記号の寄せすぎ
- アプリ上部の通貨UIをそのまま模倣する構成

## 判断基準

- 泡とまるが主役に見えるか
- 次のステージが直感的に押せるか
- 学習パス風だが、計算アプリではなく「まるをまとめるアプリ」に見えるか
- ゲーム画面でフィールドの物理的なまるが一番目立っているか
